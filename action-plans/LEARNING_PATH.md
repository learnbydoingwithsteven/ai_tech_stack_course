# AI Technology Stack - Learning Path

## 🎯 Course Overview

A structured path to mastering the complete AI technology stack, from hardware to user interfaces.

**Duration**: 6-8 weeks (8-12 hours/week)  
**Level**: Beginner to Intermediate  
**Prerequisites**: Basic programming knowledge (Python recommended)

---

## 📅 Week-by-Week Learning Path

### Week 1: Foundations
**Focus**: Understanding the big picture

#### Topics
- Module 1: Introduction to AI Technology Stack
- The five layers overview
- Use case: Drug Discovery Assistant

#### Hands-On
- [ ] Read Module 1 completely
- [ ] Map out an AI application you'd like to build
- [ ] Identify which layers are most relevant to your use case
- [ ] Document your initial design thoughts

#### Deliverable
- One-page design doc for your chosen application

---

### Week 2: Infrastructure Layer
**Focus**: Hardware and deployment decisions

#### Topics
- Module 2: Infrastructure Layer
- GPU requirements
- On-premise vs. Cloud vs. Local
- Cost analysis

#### Hands-On
- [ ] Set up a local LLM (Ollama or LM Studio)
- [ ] Run Mistral 7B or Llama 2 7B locally
- [ ] Compare response quality/speed on different hardware
- [ ] Calculate infrastructure costs for your use case

#### Deliverable
- Infrastructure decision document with cost estimates

---

### Week 3: Model Layer
**Focus**: Choosing the right model

#### Topics
- Module 3: Model Layer
- Open vs. Proprietary
- Model size considerations
- Specialization

#### Hands-On
- [ ] Test 3 different models on the same task
- [ ] Compare: GPT-3.5, Claude, and Llama 2
- [ ] Benchmark: quality, speed, cost
- [ ] Document pros/cons for your use case

#### Deliverable
- Model comparison report with recommendation

---

### Week 4: Data Layer
**Focus**: RAG and vector databases

#### Topics
- Module 4: Data Layer
- Data pipelines
- Vector databases
- RAG implementation

#### Hands-On
- [ ] Build a mini-RAG system
- [ ] Collect 10-20 documents on a topic
- [ ] Set up Chroma DB
- [ ] Implement document chunking
- [ ] Create embeddings
- [ ] Test retrieval and generation

#### Deliverable
- Working RAG prototype (Python notebook or script)

---

### Week 5: Orchestration Layer
**Focus**: Complex workflows

#### Topics
- Module 5: Orchestration Layer
- Thinking, Execution, Review
- LangChain/LangGraph
- Multi-step workflows

#### Hands-On
- [ ] Install LangChain
- [ ] Build a multi-step agent
- [ ] Implement ReAct pattern
- [ ] Add tool calling (search, calculator, etc.)
- [ ] Test with complex queries

#### Deliverable
- Orchestrated agent with 3+ tools

---

### Week 6: Application Layer
**Focus**: User experience and integrations

#### Topics
- Module 6: Application Layer
- Interface design
- Citations and revisions
- Integrations

#### Hands-On
- [ ] Build a web UI for your project
- [ ] Add text input/output
- [ ] Implement citation display
- [ ] Add follow-up question capability
- [ ] Integrate with one external tool (Slack, email, etc.)

#### Deliverable
- Functional web application

---

### Week 7-8: Final Project
**Focus**: Putting it all together

#### Topics
- Module 7: Putting It All Together
- Architecture design
- Trade-off decisions
- Deployment

#### Hands-On
- [ ] Complete your application
- [ ] Integrate all five layers
- [ ] Test thoroughly
- [ ] Deploy (at least locally)
- [ ] Create documentation
- [ ] Prepare presentation

#### Deliverable
- Complete AI application with documentation

---

## 🎯 Success Criteria

By the end of this learning path, you should be able to:

1. **Explain** all five layers of the AI stack
2. **Design** architecture for AI applications
3. **Implement** RAG systems
4. **Build** orchestrated agents
5. **Deploy** complete AI applications
6. **Make informed decisions** about trade-offs
7. **Integrate** AI into existing workflows

## 💡 Tips for Success

1. **Hands-on is key** - Don't just read, build
2. **Start simple** - MVP first, then enhance
3. **Ask questions** - Join communities, engage
4. **Document as you go** - Notes help retention
5. **Share your work** - Get feedback early
6. **Be consistent** - Regular practice > marathon sessions
7. **Experiment** - Try different approaches
8. **Have fun** - Build something you care about
